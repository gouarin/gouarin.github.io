Les logiciels autour de LBM¶

Palabos¶

  • écrit en C++
  • schémas disponibles:
    $D_2Q_9$, $D_3Q_{13}$, $D_3Q_{15}$, $D_3Q_{19}$ et $D_3Q_{27}$
  • BGK, MRT, modèle entropique, ...
  • différentes conditions aux limites:
    **Zou/He**, **Inamuro**, **bounce-back**, **périodique**, ...
  • parallélisé avec MPI

OpenLB¶

  • écrit en C++
  • schémas disponibles:
    $D_2Q_9$, $D_3Q_{13}$, $D_3Q_{15}$, $D_3Q_{19}$ et $D_3Q_{27}$
  • BGK, MRT, modèle entropique, ...
  • différentes conditions aux limites:
    **Zou/He**, **Inamuro**, **Bouzidi**, ...
  • parallélisé avec MPI et openMP

Sailfish¶

  • écrit en Python
  • schémas disponibles:
    $D_2Q_9$, $D_3Q_{13}$, $D_3Q_{15}$, $D_3Q_{19}$ et $D_3Q_{27}$
  • BGK, MRT, modèle entropique, ...
  • différentes conditions aux limites:
    **Zou/He**, **bounce-back**, ...
  • parallélisé avec CUDA ou OpenCL

Avantages¶

  • beaucoup de choix
  • codes massivement parallèles
  • codes de production

 

Inconvénients¶

  • C++ n'est pas forcément facile d'accès
  • assez rigide

Les caractéristiques de PyLBM¶

  • code écrit en Python
  • code de recherche (et de production)
  • pas de schémas prédéfinis (classique + vectoriel)
  • flexible
  • génération de codes optimisés
  • parallélisation en utilisant MPI

Idée générale¶

On écrit le schéma que l'on souhaite utiliser sous le formalisme de d'Humières.

$$ f(\mathbf{r}_i+e_\alpha \delta t, t + \delta t) - f(\mathbf{r}_i, t) = -M^{-1}\hat{S}\left[m(\mathbf{r}_i, t) - m^{eq}(\mathbf{r}_i, t)\right] $$

Informations à fournir¶

  • les vitesses,
  • les moments conservés,
  • les polynômes qui permettent d’écrire le changement de base qui diagonalise l’opérateur de relaxation,
  • les équilibres,
  • la diagonale de la matrice de collision $\hat{S}$.

Exemple¶

On souhaite résoudre une équation d'advection sur un tore à l'aide d'un $D_1Q_2$

$$ \left\{ \begin{array}{l} \partial_t u + c \partial_x u = 0 \;, t > 0,\; 0 < x < 1,\; c=1/4, \\ u(t=0,x) = u_0(x), \\ u(t,x=0) = u(t,x=1) \end{array} \right. $$

Initialisation¶

In [4]:
import matplotlib.pyplot as plt
import numpy as np
import sympy as sp
import pyLBM

X, LA, u = sp.symbols('X,LA,u')

def u0(x):
    return np.exp(-.5*(x%1-.5)**2/0.05**2)

la = 1.
Tf = 4.
c = .25
s = 1.99

Ecriture du schéma et du domaine de calcul¶

In [5]:
D1Q2 = {
    'box': {'x': [0., 1.], 'label': -1},
    'space_step': 1./128,
    'scheme_velocity': la,
    'schemes':[
        {
            'velocities': [1, 2],
            'conserved_moments': u,
            'polynomials': [1, X],
            'relaxation_parameters': [0., s],
            'equilibrium': [u, c*u],
            'init': {u:(u0,)},
        },
    ],
}

sol = pyLBM.Simulation(D1Q2) # build the simulation

Le domaine¶

Le domaine est représenté par une géométrie, un ou des schémas et un pas d'espace.

Fonctionalités¶

  • indique quels points du maillage est fluide ou solide,
  • calcule les distances pour chaque vitesse unique entre les points du maillage et le bord des obstacles,
  • fournit les labels des bords appartenant à la distance minimale calculée.

Hiérarchie des différents blocs¶

  

Génération automatique¶

Une fois les paramètres de l'utilisateur analysés, on génère automatiquement

  • le passage entre les fonctions de distributions et les moments,
  • l'équilibre,
  • le transport,
  • la relaxation.

Versions disponibles¶

  • NumPy
  • Cython
  • Pythran
  • Numba

Version Numpy¶

Les fonctions générées sont écrites de manière vectorielle et indépendante. Un pas de temps s'écrit

  • transport
  • f2m
  • relaxation
  • m2f

Les autres versions¶

On implémente l'algorithme pull

Un pas de temps s'écrit

  • on lit les données dans l'ancien f (phase de transport)
  • f2m
  • relaxation
  • m2f
  • on stocke les données dans le nouveau f

def bc_up(f, m, x, y, scheme):
    m[:, 0] = 0.
    m[:, 1] = driven_velocity
    m[:, 2] = 0.
    scheme.equilibrium(m)
    scheme.m2f(m, f)
    
mu   = 1.e-4 
zeta = 1.e-4
driven_velocity = 0.2 # velocity of the upper border
dx = 1./256
dummy = 3.0/dx
s1 = 1.0/(0.5+zeta*dummy)
s2 = 1.0/(0.5+mu*dummy)
s  = [0.,0.,0.,s1,s1,s1,s1,s2,s2]

la = 4 # velocity of the scheme

g = 1.

X, Y, LA = sp.symbols('X,Y,LA')
rho, qx, qy = sp.symbols('rho, qx, qy')

def initialization_rho(x,y):
    return rhoo * np.ones((x.size, y.size)) + deltarho * ((x**2 + y**2) < 0.25**2)

vitesse = range(1,5)
polynomes = [1, LA*X, LA*Y, X**2-Y**2]
